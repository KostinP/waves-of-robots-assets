✅ HUDController.cs
- Контроллер игрового HUD
- Работает через UI Toolkit
- Ориентирован на игровую сцену
- Принимает данные из HUDDataBridge (который, скорее всего, будет мостом между ECS и UI)
Это чистый runtime UI для игровой сцены, не для меню/настроек.

✅ HUDDataBridge.cs
- Singleton-мост между игровыми данными и UI HUD
- Хранит список игроков и их ping
- Используется HUDController
- Это UI-ориентированный MonoBehaviour, который живёт в игровой сцене
- НЕ ECS, НЕ NetCode, но получает данные от них через клиентский слой
То есть — это часть игрового UI, а не меню и не ядра.

✅ PauseMenuController.cs
Это UI, относящийся к игровому процессу, а не к главному меню. Т.е. он должен лежать рядом с HUD.
